A11y-Buddy: Your helpful web accessibility VS Code buddy!

Mission Objective


Since Accessolotls want to take over the world using accessibility, they naturally want to enable developers to make their websites accessible! What's a better way to motivate people to do something than cute virtual pets? So the Accessolotls set out to create a helpful a11y buddy to help developers fix accessibility errors while they create their websites.


Ideation


Goal: Brainstorm mechanisms to encourage and aid in developing accessible websites.

The Accessolotls spent a long time ideating on how to structure the user experience to encourage accessibility best practices. Their main ideas are outlined as follows:

Ideas on how to design the game experience including creating a tomagochi like pet with health and a rewards system, and showing users which line(s) the errors are located on.



Games used as inspiration including Forest, a study timer app where you plant trees as you study, and Habitica, which rewards you for completing habits with new armor for your character.

VS Code extensions used as inspiration including VSCode Pets, a virtual pet for your coding environment, and eslint, an accessibility checker.

Conducting Developer Interviews on Website Accessibility

The observations below were gathered from interviewing software developers on their implementation of and familiarity with website accessibility.

Observations

It is evident from the interviews that developers consider accessibility to be an afterthought and get overwhelmed by too many warnings at once using current accessibility checking extensions.

The Accessolotls were also fortunate enough to be able to gather feedback on their game idea from different classes at Brown University. That feedback is recorded below.

Takeaways

Feedback on the proposed ideas. This feedback indicated that people want a visual game with a strong emotional response. This feedback also prompted critical thought on how the extension would put the visual and technical error detection and fixing elements together.


User Flow


Below is the created user flow from reviewing the feedback provided from talking to Brown University students and developers.

User flow summary

  1. The extension determines the file type of the current open file, and if it is an html file, it generates a random egg and runs an accessibility check. If it is not an html file, it will give you a warning that it cannot run.
  2. If there are no errors, the pet's evolution bar will increase and it will say "Huzzah!". If there are errors, the pet will tell you there's at least one error. Then you can click the arrows on the extension to have each highlighted and described one at a time.
  3. Every time you open a new file or save, the accessibility tests are run. If there are no errors, your creature will grow. If there are errors, it will stop growing and look sad.

Final Working Prototype


Below are images of the final working prototype when there is and isn't currently an error.

The image shows what happens when an image is missing alt text. The pet looks sad and once you hit the forward or backward arrow, the either first or last error will be highlighted and described.





The image shows what happens when there are no errors. The buddy keeps growing and displays a Huzzah! message.

Accessolotl Analysis


An accessibility analysis of the created extension was conducted in order to determine how usable it was to beings with disabilities. Everyone deserves to Access-o-lotl!

  • VS Code does not allow audio in extension webview panels, so efforts were made to include an engaging story in the alt text. However, future research must be done on the best ways to non-visually access a visual a11y-buddy pet.
  • The error buttons staying on-screen when there are no errors could be confusing, so in the future these will be removed when there are no errors.
  • There is only a hover icon available to use to read instructions on how to use the extension, so in the future there will be a fully accessible instructions screen that does not require hovering a mouse.
  • There are only very brief descriptions of how to fix the encountered errors in the given warnings, so these will be expanded in the future to enable people who are completely new to the world of accessibility to benefit from this tool.

Mission Conclusions


After successfully creating a working prototype of a virtual pet to aid in accessible website development, the Accessolotls learned that there are still many more ways the extension itself can be more accessible, and they are excited to continue working on this project and get it published to the extension market place.
